# Патч к подсчёту по методу Шульце — что изменено

Совместимость API сохранена: `Elections`, `determine_winners`, `compute_elections`,
все флаги CLI. Это drop-in замена модулей пакета `sspmpu_elections` + папка `tests`.

## Производительность (главное)
- `get_links_order` переписан с поэлементного двойного цикла pandas (O(C⁴) по обращениям)
  на одно векторизованное вычитание numpy. Семантика {-1,0,1} та же.
- Добавлен быстрый числовой Шульце (`base_relation`/`strongest_paths`, Флойд–Уоршелл
  на numpy). `determine_winners` сначала идёт быстрым путём O(C³); тяжёлые структуры
  строятся ТОЛЬКО при ничьей.
- Стадия «иерархии случайных бюллетеней» векторизована (numpy-маски agree/disagree
  вместо C²×C² цикла на каждый бюллетень).
- Стадия «запрета звеньев» переведена на numpy-массивы (доступ по позиции) и O(1)-словарь
  сравнения звеньев вместо медленного pandas `.at[(кортеж),(кортеж)]`.
- Замеры: top-4 без ничьих — C=15 ~0.26 с (было ~9.7 с), C=20 ~0.56 с (было ~28 с).
  Полные выборы (18 кандидатов, 6 курсов + 4 общих, 60 бюллетеней) — ~3 с.

## Воспроизводимость и исправление зацикливания
- Добавлен `--seed` (и параметр `seed`); вместо незасеянного `np.random.randint`
  используется `np.random.default_rng(seed)`. В отчёт пишутся seed и № вытянутых
  бюллетеней. Один seed → один результат (проверено).
- Исправлено ПРЕДСУЩЕСТВОВАВШЕЕ зацикливание в методе запрета звеньев: `bad_link`
  всегда брался из БАЗОВОЙ матрицы путей, поэтому при некоторых конфигурациях с C≈20
  и множеством ничьих цикл не завершался (оригинал на C=20 «висит» >30 с). Добавлен
  страж завершения; неразрешённые пары попадают в финальный детерминированный тай-брейк.
- Добавлен гарантированный результат: если запрет звеньев не разрешил ничью, оставшиеся
  кандидаты упорядочиваются по иерархии бюллетеней (`candidate_hierarchy_sort`),
  а не возвращается `None`.

## Исправленные падения
- `__main__.py`: условие `if args.report_to_file is None:` было инвертировано и приводило
  к `open(None)`. Исправлено на `is not None`.
- `compute_elections`: брал `determine_winners(...)[0]` до проверки на `None` (→ `None[0]`).
  Теперь сначала проверка, потом индексация.

## Надёжность и ввод
- Кодировка отчётов `windows-1251` → `utf-8-sig` (без потери кириллицы, корректно
  открывается Excel).
- `pytz` → стандартный `zoneinfo` (с запасным вариантом).
- `pairwise_matrix` кэшируется (`cached_property`); матрицы по курсам берут кэш.
- Защита от однофамильцев (фамилия — идентификатор) и от несовпадения столбцов
  бюллетеней со списком кандидатов — понятный `ValueError`.
- Строгая проверка бюллетеней (полные строгие ранжирования) — ОПЦИОНАЛЬНА:
  `Elections(..., require_strict=True)` или флаг `--strict`. По умолчанию, как и раньше,
  допускаются равные приоритеты и пропуски.
- Убраны обратные слэши из выражений f-строк (совместимость с Python 3.10+).
- Опечатка в докстроке `ыуе` → `set`.

## Проверки (все зелёные)
- `pytest`: 11 тестов — канонический пример Шульце (→ e), свойство Кондорсе (0 провалов),
  сверка с эталоном O(C³), детерминизм, гарантия k победителей, ничья на линии отсечения,
  примеры ex5/ex11/ex12 (инварианты «содержит a, не содержит b/c»), отказ при однофамильцах,
  полный прогон выборов.
- Эквивалентность тай-брейка: векторизованная «иерархия» = прежний pandas-цикл (0 расхождений);
  numpy-«запрет звеньев» + страж = прежняя версия на 300+192 завершающихся профилях (0 расхождений).

## Замечания по существу (НЕ меняют код — на ваше решение)
- Общие места = top-k по Шульце — это КОНСЕНСУС, а не пропорциональность. Пропорциональный
  аналог — Schulze-STV.
- Идентификатор кандидата = фамилия. Сейчас стоит защита от однофамильцев; на будущее
  надёжнее ключевать по № и в `candidates.csv`, и в `ballots.csv`.
- `losses`/`ratios` при строгих полных бюллетенях совпадают с `wins`/`margins`.
